# CodeCore: страница входа (Login)

Страница авторизации CodeCore, перенесённая с макета в код. React + TypeScript + Vite.

## Превью

Двухколоночный лендинг в тёмной неоновой теме.

Слева (Hero): логотип CodeCore, заголовок с градиентным акцентом, описание, три фичи (Практика, Поддержка, Сертификаты) и неоновая SVG-иллюстрация рабочего места.

Справа (LoginCard): форма входа. В ней e-mail, пароль с переключателем видимости, «Запомнить меня», «Забыли пароль?», кнопка «Войти», вход через Google и GitHub, ссылка на регистрацию.

## Стек

- React 18 + TypeScript
- Vite 5
- Чистый CSS (без UI-библиотек)
- Иконки: шрифтовой набор [Flaticon UIcons](https://www.flaticon.com/ru/icon-fonts-most-downloaded), вариант Bold Rounded и Brands, подключается по CDN в `index.html`

## Запуск

```bash
npm install
npm run dev      # дев-сервер (http://localhost:5173)
npm run build    # продакшн-сборка в dist/
npm run preview  # предпросмотр сборки
```

```

  api/
    client.ts             HTTP-клиент (fetch-обёртка, ApiError, базовый URL)

  components/
    Hero.tsx              левая колонка: бренд, заголовок, фичи
    LoginCard.tsx         правая колонка: форма входа (стейт + вызовы API)

    icons.tsx             иконки Flaticon UIcons (Bold Rounded + Brands) как React-компоненты
```

## Интеграция с API

Слой работы с бэкендом лежит в `src/api/`. Подробные контракты с комментариями находятся прямо в коде (`client.ts`, `auth.ts`), это и есть источник правды для бэкенда.

Базовый адрес берётся из `VITE_API_BASE_URL` (см. `.env.example`), по умолчанию `/api`.

Эндпоинты, которые ожидает фронтенд:

| Метод | Путь | Назначение |
| --- | --- | --- |
| `POST` | `/auth/login` | вход по e-mail/паролю, ставит httpOnly-cookie сессии |
| `GET` | `/auth/oauth/:provider/url` | URL для OAuth (`google` \| `github`) |

Соглашения:

- Тело запросов и ответов в формате JSON. Авторизация через httpOnly-cookie (`credentials: 'include'`).
- Ошибки отдаются как `{ message, code }` с HTTP 4xx/5xx. Поле `message` (на русском) показывается пользователю как есть.
- `POST /auth/login` возвращает `{ user: { id, name, email, avatarUrl? } }`.
- Состояния запроса (loading, ошибка) `LoginCard.tsx` обрабатывает сам: блокирует форму, меняет кнопку на «Вход…», показывает блок ошибки.

## Детали реализации

- Палитра и градиенты вынесены в CSS-переменные (`:root` в `index.css`).
- Градиентный текст и кнопки сделаны через `linear-gradient` + `background-clip: text`.
- Кастомный чекбокс и кнопка «показать/скрыть пароль».
- Адаптив: при `≤980px` колонки складываются в одну, при `≤520px` соц-кнопки идут в столбик.
- Форма управляемая, сабмит вызывает `POST /auth/login` (см. раздел «Интеграция с API»).
